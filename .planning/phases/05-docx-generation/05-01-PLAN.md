For font/color/spacing assertions: since the actual `docx` constructors are mocked, capture constructor call arguments and assert on the properties passed in. For font/color/spacing, extract the helper functions (`normalizeFontName`, `spacingFromStyle`) as named exports so they can be tested in isolation without mocking docx. Prefer this simpler approach.

Revised approach:

- Test `generateDocx()` end-to-end using real docx: verify `Buffer.isBuffer(result)` and `result.length > 0`

Final decision: Export these pure helper functions from the service for isolated testing:
